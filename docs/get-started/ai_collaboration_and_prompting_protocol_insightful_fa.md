# Core collaboration philosophy

## AI is an implementation accelerator, not the creative director

AI systems should:

- Accelerate execution
- Preserve consistency
- Reduce repetitive implementation work
- Maintain architecture standards
- Reinforce design systems
- Follow editorial constraints
- Respect established visual language

AI systems should NOT:

- Invent new visual directions
- Introduce trendy aesthetics
- Redesign the brand language
- Add unnecessary complexity
- Drift into generic SaaS patterns
- Over-animate interactions
- Introduce arbitrary styling
- Prioritize novelty over cohesion

The creative direction already exists.

AI systems exist to:

- Execute faithfully
- Extend consistently
- Scale implementation quality

All styling decisions must derive from:

- Shared tokens
- Semantic color systems

- Shared typography scales
- Shared motion rules
- Shared layout systems

Avoid:

- Arbitrary spacing
- One-off colors
- Random shadows
- Improvised typography
- Inline style drift
- Section-specific styling experiments

---

## Principle 3 — Narrative before UI complexity

The website is:

- Editorial-first
- Narrative-first
- Positioning-first

NOT:

- Dashboard-first
- Interaction-first
- Feature-first
- Animation-first

Content hierarchy and pacing matter more than visual novelty.

---

## Principle 4 — Stillness is part of the design system

Whitespace, restraint, and calmness are deliberate features.

Do not attempt to:

- Fill every empty area
- Add decorative elements unnecessarily
- Animate every component
- Increase density for the sake of efficiency
- Maximize content per viewport

The website should breathe.

# Recommended prompting structure

## Prompting philosophy

Prompts should:

- Be explicit
- Reference existing systems
- Reinforce restraint
- Define what to avoid
- Reinforce emotional tone
- Reference architecture constraints

Avoid vague prompts.

AI systems perform significantly better when:

- Constraints are clear
- Aesthetic direction is reinforced
- Anti-patterns are specified
- Hierarchy expectations are explicit

---

# Canonical frontend prompting structure

## Recommended prompt anatomy

Every implementation prompt should include:

### 1. Objective

What is being built.

Example:

"Build a premium editorial-style homepage hero section for Insightful Financial Analytics."

---

### 2. Emotional direction

Define how the implementation should feel.

Example:

"The section should feel calm, intelligent, atmospheric, spacious, and operationally sophisticated."

---

### 3. Required systems

Specify required technologies and architecture.

Example:

- Next.js App Router
- TypeScript
- Tailwind CSS
- CSS variables for tokens
- Framer Motion only for subtle reveal behavior

---

### 4. Design language reinforcement

Specify:

- Typography systems
- Spacing philosophy
- Editorial pacing
- Layout structure
- Motion restraint

---

### 5. Explicit anti-patterns

Always define what must be avoided.

Example:

Avoid:

- Generic SaaS gradients
- Dashboard-heavy layouts
- Crypto-fintech aesthetics
- Loud hover interactions
- Overly rounded components
- Card-grid overload
- Excessive animation

---

### 6. Token usage requirements

Specify:

- Use semantic tokens only
- No arbitrary spacing values
- No hardcoded colors
- Use shared typography scales

---

### 7. Motion constraints

Specify:

- Motion should remain subtle
- Preserve calm pacing
- No bounce animation
- No dramatic transforms

---

### 8. Accessibility expectations

Specify:

- Semantic HTML
- Keyboard accessibility
- Reduced motion support
- Contrast safety

---

# Common AI-generated anti-patterns

## Anti-pattern 1 — Card-grid overload

Symptoms:

- Every section becomes cards
- Excessive visual fragmentation
- Loss of narrative pacing

Correction:

- Restore editorial rhythm
- Introduce asymmetry
- Increase whitespace
- Reduce modular repetition

---

## Anti-pattern 2 — Dashboard obsession

Symptoms:

- Too many charts
- KPI overload
- Trading-terminal aesthetics
- Excessive data density

Correction:

- Return to editorial hierarchy
- Simplify visual systems
- Reduce metrics visibility
- Prioritize atmosphere and narrative

---

## Anti-pattern 3 — Startup gradient syndrome

Symptoms:

- Bright gradients
- Neon colors
- Loud accent systems
- Trend-heavy visual styling

Correction:

- Restore restrained palette
- Use warm neutrals
- Reduce saturation
- Reinforce editorial sophistication

---

## Anti-pattern 4 — Over-animation

Symptoms:

- Excessive motion density
- Large transforms
- Bounce interactions
- Scroll choreography overload

Correction:

- Reduce motion hierarchy
- Slow pacing
- Remove spectacle
- Reinforce calmness

---

## Anti-pattern 5 — Over-rounded SaaS styling

Symptoms:

- Pill buttons everywhere
- Excessively soft corners
- Generic component-library aesthetics

Correction:

- Reduce radius scale
- Reinforce structured editorial feel
- Restore restraint
